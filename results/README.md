# Results

This directory contains reproducible benchmark artifacts generated by Dali evaluation runs.

Dali separates benchmark infrastructure, evaluation corpora, deterministic scoring, and generated outputs so published runs can be independently reviewed, reproduced, and compared over time.

---

## Directory structure

| Path | Purpose |
|---|---|
| `results/demo/` | Local smoke-run outputs for quick validation (gitignored) |
| `results/v0.2/` | Public benchmark artifacts for the v0.2 release |
| `results/v0.2/README.md` | Full v0.2 results: charts, leaderboard, jurisdiction breakdown, methodology |
| `results/v0.2/schema.json` | JSON Schema for per-prompt evaluation records |

---

## Quick local run

Generate a deterministic Tier 1 integrity evaluation locally (no API keys required):

```bash
python runners/run_integrity.py \
  --corpus data/public/citation_failure_cases.json \
  --output results/demo/integrity.json
```

---

## Public benchmark runs

Versioned benchmark outputs are stored under:

```
results/v0.2/<date>/
```

Each run contains per-model evaluation artifacts, methodology metadata, and policy-versioned scoring outputs. These artifacts are intended to support reproducibility, cross-model comparison, jurisdictional analysis, and independent review.

---

## Reproducibility

All benchmark outputs are generated locally from versioned corpora, deterministic scoring policies, and reproducible runners. To reproduce a full Tier 2 benchmark run:

```bash
python runners/run_synthetic.py \
  --models openai_fast openai_quality openai_production \
  --output results/v0.2/$(date +%Y-%m-%d)/
```

See [METHODOLOGY.md](../METHODOLOGY.md), [docs/policy-versioning.md](../docs/policy-versioning.md), and [results/v0.2/README.md](v0.2/README.md) for methodology, scoring policy, and run-level provenance.
