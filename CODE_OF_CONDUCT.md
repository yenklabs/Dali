# Code of Conduct

## Our standards

Dali is a technical standards project. We expect contributors to engage with
the work professionally and in good faith.

Specifically:

- Critique ideas, not people.
- Be precise. Legal and technical claims should be supported.
- Disagreement is expected and welcome. Hostility is not.
- Respect the RFC process. Circumventing review by merging unapproved spec
  changes damages the project's credibility with the legal community.

## Scope

This code of conduct applies to all project spaces: GitHub issues, PRs,
discussions, and any communications made in connection with Dali.

## Enforcement

Violations may result in removal of comments, rejection of contributions, or
blocking at the maintainer's discretion. For serious concerns, open a private
issue or contact the maintainer directly.
