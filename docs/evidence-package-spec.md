# Evidence Package Specification (EPS)

**Status:** Draft (v0.1 public summary)  
**Canonical spec:** [RFC-001: Evidence Package Specification (EPS)](../EPS/RFC-001.md)  
**Normative schema:** [Evidence JSON v1.0](specs/RFC-001-evidence-json-v1.md)

The **Evidence Package Specification (EPS)** is the interchange-format branding for portable AI evidence records — prompt, model, sources, verification outcomes, replay hashes, and sealed bundles.

[EPS/RFC-001](../EPS/RFC-001.md) defines the conceptual envelope. [Evidence JSON v1.0](specs/RFC-001-evidence-json-v1.md) defines the normative schema (`EvidenceBundle`, `CitationIntegrityResult`, replay state). The yenklabs.com draft summarizes the envelope for the public evidence ecosystem:

- [yenklabs.com/artifacts/evidence-package-spec-v0.1](https://yenklabs.com/artifacts/evidence-package-spec-v0.1)

For implementation and conformance feedback, comment on RFC-001 via GitHub issues with label `spec-change`.
