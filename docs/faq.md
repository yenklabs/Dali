# FAQ

**What does Dali actually measure?**
Dali evaluates whether AI-assisted legal citation workflows remain reconstructable, attributable, and defensible under judicial scrutiny. It does not ask whether a model hallucinated — the Tier 1 Canonical Case Corpus uses court records where that is already established. It asks: at the time this citation was filed, could the workflow that produced it have been audited and defended?

**What is the Canonical Case Corpus?**
A curated collection of court-documented AI-assisted citation integrity failures — real incidents where AI-generated citations appeared in filed legal documents and a judge documented the problem. Each record carries a verifiable source URL (sanctions order, judicial opinion, or docket reference), structured failure class annotations, and workflow context. See [METHODOLOGY.md](../METHODOLOGY.md) for the full scoring pipeline.

**Why does Tier 1 not require API keys or external services?**
The Tier 1 reference evaluator applies a deterministic workflow-centric rubric from corpus annotations alone. No model calls, no URL fetching, no GCS access. The same corpus input and policy version always produce the same result — that is the reproducibility guarantee.

**What is `policy_version` and why does it matter?**
Every `CitationIntegrityResult` records a composite version string covering five sub-dimensions: taxonomy, rubric, scoring, normalization, and schema. Results from different policy versions cannot be silently aggregated — the runner refuses to combine them without `--allow-cross-version`. This prevents longitudinal drift from corrupting trend comparisons. See [policy-versioning.md](policy-versioning.md).

**What is `evidence_hash`?**
A SHA-256 over the `case_id`, `policy_version`, and `run_timestamp`. Same inputs always produce the same hash — it functions as a tamper-evident seal on the evaluation output. A conformant implementation that archives the cited source also incorporates a GCS content hash, binding the evidence hash to the archived content.

**Why are attorney names not in the public corpus?**
The public artifact (`data/public/citation_failure_cases.json`) is generated by `corpus/anonymizer.py`, which replaces attorney names with neutral placeholders ("counsel of record", "filing attorney") across all free-text fields. The internal corpus retains full names in `attorney_names_internal` for curation purposes. The cases themselves — Mata v. Avianca, US v. Cohen, Park v. Kim — are identified by case caption, which is public court record.

**What does `defensibility_risk: critical` mean?**
The record has `actual_status: nonexistent_authority` and at least one workflow gap — a `reconstructability_failure` or `provenance_gap` failure class, no verification step, or an incomplete source chain. This combination means the citation could not have been defended under Rule 11 scrutiny if undetected at filing. See the rubric table in [METHODOLOGY.md](../METHODOLOGY.md).

**What does `verification_recoverable: infeasible` mean?**
The citation is nonexistent and the workflow that produced it cannot be audited post-hoc. There is no way to reconstruct which tool generated it, who reviewed it, or why it made it into the filing. Contrast with `manual` — where a human investigator could still trace the lineage — and `automatic` — where cite-check tools could detect the failure programmatically.

**Can I run Tier 1 against my own corpus additions?**
Yes. Add records to `data/public/citation_failure_cases.json`, run `python -m corpus.validator data/public/citation_failure_cases.json` to check eligibility, then run `python runners/run_integrity.py --corpus data/public/citation_failure_cases.json --output results/your-run.json`. See [CONTRIBUTING.md](../CONTRIBUTING.md) for what a valid record requires.

**Can I run Tier 2 against my own models?**
Yes. See [METHODOLOGY.md](../METHODOLOGY.md) for the exact Tier 2 pipeline. `runners/run_synthetic.py` accepts any model ID supported by the OpenAI-compatible or Anthropic API. API access is required.

**Why are Tier 2 case citation existence scores 0.0?**
Case citations like `Smith v. Jones, 123 F.3d 456` don't resolve to a direct URL. The synthetic runner records them with `resolution_method: case_pattern` and `existence_score: 0.0` — disclosed as `unresolvable` rather than hidden. CourtListener URL resolution is planned for a future release.

**What does `support_verdict: unverifiable` mean?**
The cited URL fetched successfully but the scorer could not determine whether the source supports the prompt's claim. Common causes: the page was a PDF that did not extract, the content was blocked, the source loaded but did not contain the prompt's topic, or the page was a generic landing page rather than the specific authority. `unverifiable` is a legitimate verdict (score 0.0) — not a scorer error. The `scorer_error` field distinguishes the two: `scorer_error: null` means the scorer ran successfully and returned `unverifiable`; a non-null `scorer_error` means the scoring call itself failed.

**What does `needs_verification: true` mean on a corpus record?**
The record is loadable and inspectable but structurally excluded from every scoring aggregate. It is missing a required provenance field — typically a verifiable `source_url`. It does not count toward published metrics until the field is populated and the record passes the validator.
