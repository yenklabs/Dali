# RFC-001: Evidence Package Specification (EPS)

**Status:** Draft  
**Version:** 0.1  
**Author:** Yen Kha  
**Project:** [Dali](https://github.com/yenklabs/Dali)

> This RFC defines the conceptual envelope for portable AI evidence. The normative v1.0 schema for citation integrity results is [Evidence JSON v1.0](../docs/specs/RFC-001-evidence-json-v1.md). Comments and conformance feedback are welcome via GitHub issues with label `spec-change`.

---

## Summary

Current AI systems produce outputs but rarely preserve enough evidence to independently reconstruct, audit, or verify how those outputs were generated. This RFC proposes the **Evidence Package Specification (EPS)**, an open, portable format for representing AI-generated evidence across models, vendors, and applications.

Rather than standardizing model behavior, EPS standardizes the evidence required to evaluate and reproduce model-assisted work.

Legal AI is Dali's current proving ground — court-documented citation failures, deterministic integrity scoring, and reproducible benchmark runs — but EPS is intended to generalize to other high-stakes domains over time.

---

## Motivation

Professionals increasingly remain accountable for AI-assisted decisions in domains such as law, healthcare, finance, and public administration. Existing AI systems generally preserve only prompts and outputs, leaving little ability to reconstruct the reasoning process or verify the supporting evidence months later.

This creates a governance gap:

- AI outputs cannot be independently reproduced.
- Evidence chains are fragmented across vendors.
- Human review is difficult to reconstruct.
- Regulators cannot reliably audit AI-assisted decisions.

EPS addresses this problem by defining a portable evidence format. Dali's open repository demonstrates the approach today on citation integrity: every Tier 1 run produces a policy-versioned, hash-sealed `CitationIntegrityResult` that CI re-verifies for replay equality on every pull request.

---

## Goals

EPS aims to provide a common evidence representation that is:

- **Portable** across AI platforms
- **Vendor-neutral**
- **Cryptographically verifiable** — tamper-evident hashes over inputs, policy, and run metadata (see [cryptographic lineage](../docs/cryptographic-lineage.md))
- **Deterministically replayable** — stable inputs under a fixed policy version produce stable verdicts (Tier 1 today; Tier 2 preserves methodology metadata for comparable reruns)
- **Human review compatible** — workflow context (retrieval use, review steps) is part of the evaluation rubric even where a formal review-action schema is still evolving
- **Suitable for long-term audit and governance**

## Core Components

Every Evidence Package should preserve enough structure for an independent reviewer to reconstruct how an AI-assisted conclusion was produced. EPS v0.1 groups these into the following components:

| Component | v0.1 status in Dali | Notes |
|---|---|---|
| Source documents | Implemented | Tier 1 corpus records; Tier 2 prompt corpora in [`data/benchmark/`](../data/benchmark/) |
| Retrieved authorities | Implemented | URL resolution, existence checks, archived content hashes |
| Canonical identifiers | Implemented | CourtListener IDs, jurisdiction codes, authority types (see [Evidence JSON §4](../docs/specs/RFC-001-evidence-json-v1.md#4-taxonomies)) |
| Model inputs | Partial | Tier 2 synthetic probes; full bundle envelope in progress |
| Model outputs | Implemented | Per-prompt Tier 2 result artifacts in [`data/results/`](../data/results/) |
| Verification results | Implemented | `CitationIntegrityResult` per cited source |
| Human review actions | Planned | Workflow context captured in methodology; formal schema reserved for a later RFC |
| Policy versions | Implemented | Composite `policy_version` over taxonomy, rubric, scoring, normalization, and schema (see [policy versioning](../docs/policy-versioning.md)) |
| Cryptographic hashes | Implemented | `corpus_record_hash`, `replay_hash`, and `evidence_hash` on every integrity result |
| Provenance metadata | Implemented | Run methodology JSON, replay state dimensions, lineage between Tier 1 cases and Tier 2 derivatives |

Together, these components allow an independent reviewer to ask not only *whether a citation exists*, but *whether the workflow that produced it can be audited and defended under a fixed policy version*.

The v1.0 interchange container is the **`EvidenceBundle`**: a top-level envelope with `CitationIntegrityResult` records, `replay_state`, and aggregate summary statistics. Machine-readable schemas live in [`dali/schemas/`](../dali/schemas/).

## Why This Matters

Most AI evaluation measures capability.

EPS measures accountability.

Instead of asking:

> Can the model generate a convincing answer?

EPS asks:

> Can an independent reviewer reconstruct and verify how that answer was produced?

This distinction becomes increasingly important wherever humans remain legally or professionally responsible for AI-assisted work — from sanctions orders involving fabricated citations ([case studies](../docs/CASE-STUDIES.md)) to future regulated domains where provenance is mandatory.

## Current Status

The [Dali](https://github.com/yenklabs/Dali) project provides the first reference implementation of EPS, focused on legal citation integrity.

**Implemented in the open repository today:**

- **Deterministic replay** — Tier 1 evaluator runs offline; `--verify-replay` and [CI replay verification](../.github/workflows/replay-verification.yml) enforce byte-identical `replay_hash` values
- **Policy versioning** — composite sub-versions block silent longitudinal drift
- **Cryptographic evidence hashes** — three-hash lineage chain on every result
- **Citation verification** — existence scoring, authority classification, and URL recoverability
- **Provenance tracking** — methodology metadata, corpus lineage, and replay state dimensions
- **Reproducible benchmark datasets** — public corpora on [Hugging Face](https://huggingface.co/yenklabs) and versioned runs in [`data/results/v0.2/`](../data/results/v0.2/)

**In progress (v1.0 freeze work):**

- Full `EvidenceBundle` emission from the reference runner (schema defined; envelope wrapping tracked in [roadmap](../docs/roadmap.md))
- Evidence JSON v1.0 RFC publication and conformance tooling
- Expanded cross-jurisdiction corpus and semantic support scoring (v3 roadmap)

**Hosted product layer:**

[Dali Platform](https://dali.gammalex.com) operationalizes sealed evidence records, MCP/API audit hooks, and exportable artifacts at matter scale. The open repo remains the reproducibility anchor; the platform extends EPS into live review workflows. See the [platform MCP contract](../docs/platform-mcp-contract.md) for the public audit envelope.

Planned areas include:
